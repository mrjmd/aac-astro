**Website Tactics:**
1. **Detailed technical explanations** - Don't dumb it down too much

4. **Awards/recognition** - BBB, Angi, HomeAdvisor badges

3. **Privacy policy, terms of service**
4. **Secure payment options** (if applicable)

## Part 6: AI Overview Optimization

### The New Search Landscape

> "Google AI Overviews now appear in 50%+ of search results. Businesses that regularly post updates or add reviews are seen as active and trustworthy."

### How to Rank in AI Overviews

**1. Entity Reinforcement**
AI systems rank business entities, not just pages. Your business must appear consistently across:
- Website
- Google Business Profile
- Citations
- Reviews
- Social media

When AI encounters the same entity described consistently across multiple independent sources, it builds confidence.

**2. Question-Based Content**
Question searches are 84% more likely to trigger AI Overviews.

**Create content answering:**
- "How much does foundation crack repair cost?"
- "What causes cracks in foundation walls?"
- "How long does foundation repair take?"
- "Is foundation crack repair worth it?"
- "When should I worry about foundation cracks?"

**3. Schema Markup**
AI Overviews pull from structured data. Attack A Crack already has:
- LocalBusiness schema ✓
- FAQPage schema ✓
- AggregateRating schema ✓
- BreadcrumbList schema ✓

**Add:**
- HowTo schema on service pages (process explanations)
- Article schema on blog posts
- VideoObject schema (when video content is added)

**4. Direct Answer Format**
Structure content so the core answer appears in the first sentence (BLUF Method - Bottom Line Up Front).

**Example:**
```
Bad: "Foundation crack repair is a complex process that depends on many factors..."

Good: "Foundation crack repair typically costs $500-$3,000 for most cracks,
with the average homeowner paying around $1,500. The exact cost depends on..."
```

**5. Review Volume & Quality**
AI Overviews favor businesses with strong review profiles:
- 4.5+ star rating (4.0 minimum)
- 20+ recent reviews
- Active review responses

## Part 7: Review Strategy

### The 2026 Review Reality

> "A 4.5+ average with at least 20 recent reviews and active engagement is sufficient. A 'perfect' 5.0 with zero negative feedback is often flagged as suspicious by AI filters."

Reviews now influence:
- Local pack rankings
- AI Overview inclusion
- Click-through rates
- Conversion rates

### Review Acquisition Strategy

**Target:** 2-4 new Google reviews per month

**Process:**
1. **Ask at the right time** - After successful job completion
2. **Make it easy** - Provide direct link to Google review page
3. **Follow up** - Send reminder email/text if needed
4. **Ask for specific feedback** - "Could you mention what service we did?"

**Review Request Template:**
```
Hi [Name],

Thank you for choosing Attack A Crack for your foundation repair!
We're glad we could help.

If you have a moment, would you mind sharing your experience on Google?
It really helps other homeowners find us.

[Direct link to Google review]

Thanks again,
[Your name]
```

### Review Response Strategy

**Respond to ALL reviews within 24 hours**

AI now summarizes reviews for users. Your responses affect how AI perceives you.

**Positive Review Response:**
- Thank them specifically
- Mention the service (reinforces keywords)
- Personalize if possible

**Example:**
```
Thank you, [Name]! We're so glad the foundation crack repair
at your Cambridge home went smoothly. It was a pleasure working
with you. Don't hesitate to reach out if you ever need anything else!
```

**Negative Review Response:**
- Acknowledge the concern
- Apologize for the experience
- Take it offline ("Please call us at...")
- Show you're fixing it

AI evaluates how you respond to criticism.

### Review Platform Priority

1. **Google** (primary - affects rankings)
2. **Yelp** (secondary - high trust)
3. **BBB** (tertiary - authority signal)

## Part 11: City Page Uniqueness Strategy

### The Thin Content Risk

With 155+ city pages planned, duplicate/thin content is a serious risk. Google may:
- Not index similar pages
- Consolidate into a single result
- Penalize for doorway pages

### Uniqueness Requirements per City Page

Each city page MUST have unique:

| Element | How to Achieve | Example |
|---------|---------------|---------|
| Title | City + Service focus | "Foundation Repair in Cambridge, MA" |
| Meta description | City-specific pain point | "Cambridge's aging brownstones need..." |
| H1 | Unique positioning | "Cambridge Foundation Specialists" |
| Opening paragraph | Local context | "From Harvard Square to Inman Square..." |
| Neighborhoods section | Real neighborhoods | "Porter Square, Kendall Square, etc." |
| Landmarks | Local landmarks | "Near MIT, Fresh Pond Reservation" |
| Foundation types | Common in that city | "Cambridge's brick rowhouses often have..." |
| Customer testimonial | From that city (if available) | "Great service!" - John, Cambridge |
| FAQ | City-specific questions | "How much does foundation repair cost in Cambridge?" |
| CTA | Local phone/action | "Call us for Cambridge foundation inspection" |

### Content Template Variables

Each city content file should include:
```yaml
neighborhoods:
  - Porter Square
  - Harvard Square
  - Kendall Square
localLandmarks:
  - MIT
  - Harvard University
  - Fresh Pond
commonFoundationTypes:
  - Brick
  - Stone
averageHomeAge: "1890-1920"
localChallenges: "Cambridge's proximity to the Charles River creates unique moisture challenges..."
```

### Automated Uniqueness Checks

Add to CI/CD validation:
```bash
# Check for duplicate descriptions
npm run validate:uniqueness
# Fails if >80% content similarity between city pages
```

## Part 12: Multi-Hub Entity Schema

### The subOrganization Strategy

For businesses serving multiple regions, Google recommends establishing entity relationships:

```json
{
  "@type": "LocalBusiness",
  "name": "Attack A Crack Foundation Repair",
  "url": "https://attackacrack.com",
  "areaServed": [
    { "@type": "State", "name": "Massachusetts" },
    { "@type": "State", "name": "Connecticut" },
    { "@type": "State", "name": "Rhode Island" },
    { "@type": "State", "name": "New Hampshire" },
    { "@type": "State", "name": "Maine" }
  ],
  "subOrganization": [
    {
      "@type": "LocalBusiness",
      "name": "Attack A Crack - Boston Metro",
      "areaServed": { "@type": "City", "name": "Boston" },
      "url": "https://attackacrack.com/massachusetts/boston"
    },
    {
      "@type": "LocalBusiness",
      "name": "Attack A Crack - Hartford Area",
      "areaServed": { "@type": "City", "name": "Hartford" },
      "url": "https://attackacrack.com/connecticut/hartford"
    }
  ]
}
```

**When to Implement:** Post-launch, once 20+ city pages exist. Premature implementation with few pages looks thin.
